前端手写合集

防抖

函数防抖就是在函数需要频繁触发的情况下，只有足够的空闲时间，才执行一次。

典型应用

  • 搜索框在输入稍有停顿时才更新推荐热词
  • 拖拽

节流

一个周期内只执行一次。好像水滴积攒到一定程度才会触发一次下落一样。

典型应用：

  • 抢券时疯狂点击，既要限制次数，又要保证先点先发出请求
  • 窗口调整
  • 页面滚动

手写apply(call)

  • 首先要在原型上编程即Function.prototype
  • 拿到函数的引用，即this
  • 传入对象.fn = this
  • 执行传入对象.fn(传入参数)
  • 返回执行结果

深拷贝

简单方法

JSON.parse(JSON.stringfy)

缺点：

  • 会忽略undefined
  • 会忽略symbol
  • 会忽略函数
  • 不能解决循环引用的对象 （会抱错）

封装深拷贝

思路：

  1. 使用for-in遍历对象。
  2. 因为for-in会遍历原型链上的属性，所以要判断属性是否在原型链上，不在原型链上才可以拷贝。
  3. 判断属性值是原始类型还是引用类型。
  4. 原始类型直接赋值（注意null）。
  5. 引用类型判断是对象还是数组，创建对应的空对象或者空数组，调用递归函数，将值赋进去

选择排序

  • 在未排序序列中找到最小（大）元素，存放到排序序列的起始位置。
  • 从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。
  • 重复第二步，直到所有元素均排序完毕。

快速排序

  • 先从数列中取出一个数作为“基准”。
  • 分区过程：将比这个“基准”大的数全放到“基准”的右边，小于或等于“基准”的数全放到“基准”的左边。
  • 再对左右区间重复第二步，直到各区间只有一个数。
